Legal

Informational use not investment advice No guarantees future outcomes vary Production posture secure deployment matters

Important product disclosures, access conditions, and responsibility boundaries.

This page explains how the platform should be interpreted in a professional setting. It is not a return guarantee and not a substitute for regulated documentation.

Core Notices

The interface may present investment-style information, but it still must be read responsibly.
  • Displayed gains, losses, and percentages are portfolio indicators, not promises of future outcomes.
  • Admins and users remain responsible for validating records and exported materials.
  • Public pages communicate the product. Secured routes handle the real operational data.

System Boundary

Authenticated portfolio information belongs behind the backend, not inside public pages.

The public website explains the platform. Holdings, settings, and auth decisions belong to secured routes.

Development Note

Testing features such as OTP preview mode should be disabled before production release.

Real deployments should use external delivery providers and server-side secret management.

Recommended Production Posture

What should be true before a client-facing launch.

A premium public design works best when the operational setup is just as disciplined.

01Real OTP delivery

Replace preview mode with SMS or another production-grade verification channel.

02Secrets hardening

Keep database, auth, and provider keys entirely outside the browser.

03Retention policy

Define how reviews, exports, audit entries, and activity data are handled over time.

04Client documentation

Provide a clear ownership handover for hosting, domains, database, and operations.

Clear Boundaries Matter

Professional products are not only attractive. They are explicit about responsibility.

This legal layer reinforces credibility by making usage conditions and system boundaries easy to understand.
